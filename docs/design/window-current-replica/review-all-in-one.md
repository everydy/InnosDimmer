# Control Window Replica Review

Date: 2026-06-23

## Scope

Review target: `docs/design/window-current-replica/control-window-replica.html`

Evidence target: native AppKit safe-smoke screenshots generated by:

```bash
xcodebuild test -scheme InnosDimmer -only-testing:InnosDimmerTests/MenuBarStateTests/testUnifiedAppWindowSafeVisualSmokeRendersNonblankPages CODE_SIGNING_ALLOWED=NO
```

## Findings

### P1: Replica needs a browser-visible app-window frame

Native evidence files under `/tmp/InnosDimmerSafeSmoke/` are 1800x1280 Retina screenshots of a 900x640 `contentView`. They do not include the macOS traffic-light titlebar.

However, the user-facing replica is reviewed in a browser as a window mock. The replica should therefore keep a fake app-window titlebar and outer rounded frame, while preserving the 900x640 native content area below that titlebar.

### P1: Hidden header controls were still visible

The replica used `hidden` on schedule and diagnostics header controls, but `.button { display: inline-flex; }` overrode the browser hidden behavior for button elements. This made `Next 09:00` and `Export diagnostics` appear on the Overview page even though native Overview only shows `Paused` and `Login item on`.

### P1: Overview values were stale

Native Overview shows:

- `Paused`
- `Login item on`
- Quick actions chip `Manual`
- `Resume automation`
- Status schedule `09:00 · 80% / warmth 12%`
- Diagnostics `clear`

The replica showed automation-active/manual mismatches, `Pause automation`, `09:00 · 100% / warmth 0%`, and `1 warning(s)`.

### P2: Display page used the wrong field model

Native Display uses a `CURRENT STATE` table plus a target display section with a native pop-up control and row fields. The replica used stale names, `Display 2`, external-only safety text, and boxed summary tables where the native page uses flatter field rows.

### P2: Schedule page action and state did not match paused automation

Native Schedule shows `Next 19:00`, `Paused until 19:00`, and bottom actions `Resume automation` and `Save schedule`. The replica showed `Next 09:00`, `Active`, and `Pause automation`.

### P2: Diagnostics page used old log and summary content

Native Diagnostics shows `Verification: 0/10 handled · handled checks · 0 blocked` and a single safe-smoke log line. The replica kept older multi-line diagnostic sample data.

## Required Fix Direction

- Treat the HTML as a capture replica, not as a product redesign.
- Preserve the native 900x640 contentView model inside a browser-visible fake app-window frame.
- Keep the current comparison page URLs stable.
- Prefer exact native text values over ideal mockup text values.
